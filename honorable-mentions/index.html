Honorable Mentions

These are well known entries in the genre of people refusing to fizz-buzz normally. They are a wonderful inspiration but are languages and platforms (a type checker, a neural net, a corporate codebase) that don't lend themselves to an in-browser reenactment. Read the originals below.

FizzBuzz in TypeScript's Type System gal.hagever.com

No runtime code at all — the "program" lives entirely in type declarations. Peano numbers are built from nested tuples, Increment/Decrement/ Subtract are conditional types with an infer-based unwrap trick, and divisibility falls out of recursive subtraction. A passing test is a successful tsc compilation; a failing one is a type error. The "runtime" is the TypeScript compiler itself.

Fizz Buzz in TensorFlow joelgrus.com

Joel Grus's classic: instead of writing the mod-3/mod-5 logic, train a neural network on the binary representation of numbers and their fizzbuzz labels, then let gradient descent approximate the function. It "works," mostly, which is the joke — a satire of throwing machine learning at problems that don't need it, years before that became a whole genre.

FizzBuzz Enterprise Edition github.com

The cursed one. A fully "enterprise" Java implementation of FizzBuzz complete with abstract factories, strategy patterns, dependency injection, and a class hierarchy several stories tall — all in service of printing sixteen lines of text. A working satire of enterprise Java architecture that is funnier the longer you scroll through the source tree.
