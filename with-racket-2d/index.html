FizzBuzz as a 2D Table

In Racket, syntax is whatever the reader says it is — and the 2d language's reader accepts a literal table drawn in Unicode box characters. #2dmatch is a two-dimensional match: the top-left cell names two expressions, the row and column headers are patterns for them, and the program's decision table is its decision table. FizzBuzz's whole shape — two divisibility tests, four outcomes — drawn as what it always was: a 2×2 grid. 

#lang 2d racket
(require 2d/match)
(define (fizzbuzz n)
  #2dmatch
  ╔═══════════════════════════╦══════════════╦════════════════════╗
  ║ (modulo n 5) (modulo n 3) ║      0       ║         _          ║
  ╠═══════════════════════════╬══════════════╬════════════════════╣
  ║         0                 ║  "FizzBuzz"  ║       "Buzz"       ║
  ╠═══════════════════════════╬══════════════╬════════════════════╣
  ║         _                 ║    "Fizz"    ║ (number->string n) ║
  ╚═══════════════════════════╩══════════════╩════════════════════╝)

(for ([n (in-range 1 101)])
  (displayln (fizzbuzz n)))

(modulo n 5) picks the row (modulo n 3) picks the column _ matches anything

The real thing, actually run

No simulation this time. This is Minimal Racket in an Alpine container (the Dockerfile is 20 lines, the image is 188 MB) reading that table and printing: 

$ docker build -t fizzbuzz-racket-2d .
$ docker run --rm fizzbuzz-racket-2d

Source: this exact table was posted to r/Racket as “the first beautiful implementation of FizzBuzz”. The 2d reader really does parse the box drawing — misalign one ║ and it's a syntax error.
